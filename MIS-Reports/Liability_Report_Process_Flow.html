Core Primary Roles

  • Data Hub: This function is the monolith orchestrator for extracting multi-dimensional payroll liability data including Billing, Ledgers, Meal Vouchers, and Taxes.
  • Polyfill Nature: Depending purely on p_action and p_reporttype parameters, the same function manipulates the base tables to return either high-level summaries or nested JSON employee-level ledger properties.
  • Multi-Cursor Architecture: Dynamically opens and returns different `refcursor` variables (e.g., v_rfc_disbursment, v_rfc) based on the execution path.

Logic Key Business Rules

  • Dynamic Constraints: Employs Geo-Fencing, Organization Unit mapping, and department/designation filters using parameter arrays evaluated against openappointments properties.
  • Ledger & Deduction Separation: Sub-queries utilize tbl_employeeledger to parse out taxable ledgers, loans, and non-taxable components natively converting them into JSON objects on-the-fly (`json_agg`).
  • Arrears Handing: Safely computes arrears vs current wages by cross-referencing record screens like Increment Arear and safely mapping arrears back to the active tracking grids.
