Empirical Database Schema Analysis

A definitive architecture visualizer constructed directly from PostgreSQL empirical row-counts. This enhanced version strictly details the actual enumerated Master Values governing logical flow inside the database engine.

01

HRMS & Master Data

Static configuration for employees and corporate funding entities.

openappointments

  • emp_code VARCHAR
    Unique identifier across the system. (Top tier foreign key driver).
  • appointment_status_id INT
    Governs exactly where an employee is within the onboarding lifecycle. Fully verified setups are marked as '11'.
    11 = Converted (Active) 1 = Incomplete Onboarding 13 = Cancelled 15 = Suspended

employer

  • account_id BIGINT
    Links physical profiles to internal virtual wallets in tbl_account.
  • ach_transaction_type VARCHAR
    Global remitter configuration for banking logic routing.

empsalaryregister

  • gross / ctc / basic NUMERIC
    Contractual numeric margins holding the final max bounds scaled by Loss-of-Pay multipliers.
02

Unprocessed Inputs & Staging

Timesheets and Bio-device aggregations moving towards calculations.

tbl_monthly_attendance

  • approval_status VARCHAR
    Records the specific administrative validation state of daily biometric/manual punches.
    A = Approved P = Pending

cmsdownloadedwages

  • attendancemode VARCHAR
    Describes the exact channel the aggregated billable input originated from before hitting staging.
    Ledger (Automated rollup) MPR (Manual File Feed) Manual (Admin Override)
  • issalaryunlocked BIT
    Validation barrier. NULL implies it has been locked out from any processing pipeline execution.
03

Core Payroll & Billing Engine

The Nexus where execution happens defining final mathematical balances.

tbl_monthlysalary

Architectural Fact: The final payroll block natively combines tracking for physical payouts alongside internal accounting bounds! By reading issalaryorliability, the system parses the ledger row identity.
  • issalaryorliability VARCHAR
    Hard-stops the row designation. Identifies if the numeric netpay represents cash owed vs accounting burden.
    S = Salary Equivalent L = Tax/Compliance Liability
  • issalarydownloaded VARCHAR
    The critical payout flag to safely identify batches without sending duplicates to banks.
    P = Processing Finalized/Paid H = Held NULL = Pending Disbursement
  • batchid VARCHAR
    Group execution tag allowing mass commits or rollbacks on bulk reports.

managetemppausedsalary

  • pausedstatus VARCHAR
    Administrative override actively stopping the employee from passing to the banking engine.
    Enable = Actively Blocked Disable = Resumed
