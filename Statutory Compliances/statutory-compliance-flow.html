Statutory Compliance Process Flow

This diagram illustrates the data flow for EPF, ESIC, and LWF compliance reporting.

flowchart TD Start([User Requests Compliance Report
EPF / ESIC / LWF]) --> ActionChoice{Which Action is Requested?} %% Action Paths ActionChoice -- "DisplayReport" --> PrepDisplay[Prepare Query for Displaying Data] ActionChoice -- "DownloadReport" --> PrepDownload[Prepare Query for Inserting & Returning Data] ActionChoice -- "RetrieveProcessedReport" --> FetchProcessed[Fetch Already Processed Data] ActionChoice -- "Demark" --> MarkDownloaded[Mark Challan as Downloaded] ActionChoice -- "DisplayArrearChallan" --> PrepArrear[Prepare Arrear Challan Query] %% 1. DisplayReport & DownloadReport Logic PrepDisplay & PrepDownload --> DataSources[(Data Sources)] DataSources -.-> MasterEmp(OpenAppointments
Master Employee Info, UAN, ESIC No.) DataSources -.-> MonthlySal(tbl_monthlysalary
Current Monthly Wages) DataSources -.-> ArrearSal(tbl_monthly_liability_salary
Arear Salary & Adjustments) DataSources --> FilterData[Filter Data] FilterData --> F1[Filter by Month & Year] FilterData --> F2[Filter by Valid Org Unit / IP] FilterData --> F3[Exclude rejected/invalid wages] FilterData --> F4[Filter by Report Type
EPF > 0, ESIC > 0, LWF > 0] F4 --> CombineData[Combine and Aggregate Data] CombineData --> MathCalc(Mathematical Computations) MathCalc -.-> C1[Calc Gross, EPF, EPS, EDLI Wages] MathCalc -.-> C2[Enforce Maximum Thresholds
e.g. EPF/EPS limit of 15,000] MathCalc -.-> C3[Calculate Difference EPF - EPS] MathCalc -.-> C4[Calculate NCP Days
Month Days - Paid Days] MathCalc --> DisplayResult{Is Action Download?} DisplayResult -- "No" --> RetDisplay([Return Calculated Report for Screen]) %% 2. Download Report Specific Logic DisplayResult -- "Yes" --> InsertStaging[Insert new Data into
epfecrreport table] InsertStaging --> RetDownload([Return Calculated Report for Download]) %% 3. Retrieve Processed Report Logic FetchProcessed --> ProcessedTable(epfecrreport) ProcessedTable --> JoinMaster[Join with OpenAppointments] JoinMaster --> RetProcessed([Return Final Processed Data]) %% 4. Demark Logic MarkDownloaded --> InsertChallanDate[Insert into tblpfchalandates
indicating user downloaded the challan] InsertChallanDate --> RetNull([Return Null/Success])
