Core Payroll Database Tables Dictionary

A reference guide to the most critical tables, architectures, and columns governing the end-to-end payroll generation flow.

openappointments

The master foundation table representing employee deployment records. All payroll and attendance calculations pivot around records populated here from the CRM/HR system (HUBTPCRM).
Column Name Data Type Business Significance
emp_code PK VARCHAR Unique employee identifier used across all systems to track payroll, leaves, and bank transfers.
customeraccountid FK BIGINT Links the employee to their respective employer/client limits and funding (tbl_account).
appointment_status_id INT Must be '11' indicating an active onboarding/deployment state ready for payout routing.
converted VARCHAR(1) Flags whether the candidate successfully converted to an active employee status (value 'Y').

cmsdownloadedwages

A highly transactional staging table populated by the bulk attendance approval module. Acts as the transition zone between raw timesheets and calculated salary figures.
Column Name Data Type Business Significance
empcode VARCHAR Maps attendance records back to the employee.
billable_days NUMERIC Net days the employee worked and expects to log payment for (essential for Prorata logic).
loss_of_pay_days NUMERIC Leaves or absences that trigger a gross amount deduction during the wage generation procedures.

tbl_employeeledger

Central tracking table for variable/custom salary components including allowances, performance bonuses, or ad-hoc deductions (e.g. damages, missing assets).
Column Name Data Type Business Significance
emp_code VARCHAR Employee against whom the ledger entry is drawn.
amount NUMERIC The exact financial value to add or deduct from standard earnings.
payheadtype VARCHAR Defines component behavior: e.g. "Allowance" (increases NetPay) vs "Deduction" (decreases NetPay).

tbl_monthlysalary

The massive monolithic master table for payroll storage. Every processed month creates one row per employee comprising all statutory, gross, deduction, and arrears calculation traces. Highly critical for salary slips and banking.
Column Name Data Type Business Significance
batchid VARCHAR Unique tag uniting all records processed simultaneously in a single UI click. Used for isolated commit/rollback.
mprmonth / mpryear INT The exact calendar matrix block (Target Month/Year) the salary line belongs to.
netpay NUMERIC The final calculated payout (Earnings - Deductions). Determines how much goes to the Bank string.
issalarydownloaded VARCHAR Crucial flag ('P' = Paid, 'N' = Unpaid) telling systems whether HDFC / HSBC disbursement has already occurred. Prevents double-payout loops.
issalaryorliability VARCHAR Flag ('S' = Salary, 'L' = Liability) specifying if this represents direct cash payout or an indirect tax/client obligation overhead booking.

banktransfers

Stores historical records of payments sent out physically through Bank Integrations. Linked directly to the employer payout engine.
Column Name Data Type Business Significance
bankbatchcode VARCHAR HDFC / HSBC structured batch code combining "SALAKL" and month tags. Identifies grouping in the physical CSV output.
salary NUMERIC Matches `tbl_monthlysalary.netpay`. Required for auditing anomalies.
salmonth / salyear INT Maps the payment record back to the respective monthly billing run.

tbl_account (Employers)

Administers overarching customer/employer rules regarding available wallet balances and payout pacing routines.
Column Name Data Type Business Significance
id PK BIGINT Employer reference key used to trace aggregated fund allocations.
payout_mode_type VARCHAR 'standard' ensures payouts wait for sufficient balance; others bypassed rules inside uspemployerbankpayout.
payout_period VARCHAR 'Advance' signals the engine to release salaries prior to standard month-end dates.

tbl_monthly_attendance

Captures the unaggregated, raw daily physical/digital punch data. Forms the foundational input for cmsdownloadedwages generation.
Column Name Data Type Business Significance
emp_code VARCHAR The employee being tracked.
attendance_date DATE The specific calendar date of the punch.
status VARCHAR Status of the day (e.g., 'Present', 'Absent', 'Half Day', 'Leave').

tbl_receivables

Manages the wallet and physical cash-flow for customer accounts. Payout algorithms (uspemployerbankpayout) use this to check if a client has sufficient credits before releasing bank transfers.
Column Name Data Type Business Significance
customeraccountid FK BIGINT Maps back to the specific employer.
entrytype VARCHAR Defines if this is a 'Receipt' (cash added) or 'Invoice' (liability registered).
netamount NUMERIC The financial quantum added or deducted from the customer's rolling wallet balance.

managetemppausedsalary

Administrative override table to intentionally pause/block salaries from hitting the bank transfer batches, regardless of calculation success.
Column Name Data Type Business Significance
empid BIGINT The targeted employee whose payout is halted.
processmonth / processyear INT The specific calendar run locked. Null values may imply an indefinite block depending on status.
pausedstatus VARCHAR Flag ('Enable' / 'Disable') evaluated inside uspemployerbankpayout during payout staging.

tbl_employer_challan_deposit

Captures statutory regulatory deposits performed by the employer for things like PF and ESIC. This governs if statutory obligations reflect as valid liabilities in calculations.
Column Name Data Type Business Significance
customeraccountid FK BIGINT The employer remitting the funds.
challantype VARCHAR Type of the deposit ('PF' or 'ESIC').
totalchallanamount NUMERIC The monetary equivalent deposited, aggregated into functions for limit verification.
