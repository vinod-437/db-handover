Here is a step-by-step description of what is happening in the flowchart (and by extension, the stored procedure).

The flowchart is broken down into three main phases: Setup & Validation, Branching Logic, and Final Computations.

Phase 1: Setup & Validation (The Top Section)

  • Fetch Deductions & Settings: The procedure starts by pulling static configurations. It checks if the employee has specific add-ons like Meal Vouchers, Medical Insurance, National Pension System (NPS), or Government Bonus opted in. It also checks employer-level settings (like whether Provident Fund (PF) caps are strictly applied).
  • Minimum Wages Check: The system verifies if strict Minimum Wage compliance is turned on (usually for companies with >20 employees where the user has opted out of PF).
  • Validation: If this rule applies, the code enforces a hard block: the Basic salary cannot be ≤ 15,000, and the total CTC cannot be < 17,440. If either fails, the procedure stops immediately and throws an error.

Phase 2: Salary Structure Branching (The Middle Section)

The procedure diverges into two paths depending on the assigned Salary Category ID.

Path A: Category 1 or 2 (Fixed/Metro Calculations)

This path is usually for management or fixed-structure employees.

  • Base Calculation: It calculates exact Base HRA (usually 40% or 50% depending on Metro/Non-Metro) and Base Allowances directly off the Basic salary.
  • PF/NPS Logic: Computes the exact EPF and NPS based on the Basic.
  • ESI Back-Calculation: If the employee falls into the ESI range (Employee State Insurance), the procedure must find the true Gross Salary. Because ESI charges the employer 3.25% of the Gross, it takes the Target CTC, subtracts fixed deductions (PF, Bonus, LWF), and divides the remaining amount by 1.0325 to find the exact, mathematically perfect Gross.
  • Allowance Fix: Finally, it shrinks or inflates the HRA and Allowances to tightly match that exact Gross.

Path B: Other Categories (Dynamic Build-Up)

This path dynamically juggles variables to ensure the final CTC exactly matches the offered target.

  • Subtract Variables: It temporarily removes performance variables and non-wage items from the target CTC.
  • ESI Check: Checks if the currently projected Gross is ≤ ₹21,000 (the standard Indian threshold). If it is, ESI (3.25% for the Employer, 0.75% for the Employee) is activated.
  • The Balancing Point (“Calculated CTC vs Target CTC”): It sums up everything calculated so far to create a "Calculated CTC".
  • If Calculated < Target: It increases the Allowances by the difference (adjusting by ÷ 1.0325 to compensate if ESI is silently adding tax overhead).
  • If Calculated > Target: It safely shrinks the Allowances using the exact same logic.

It repeats this until the Gross is balanced perfectly with the target setup.

Phase 3: Final Computations (The Bottom Section)

Once the main mathematical juggling in Phase 2 is done, both paths converge back together to finalize the structure:

  • Professional Tax (PT): Since PT relies on the final complete Gross, it looks at the assigned state's tax slabs and deducts the appropriate Professional Tax amount based on the finalized Gross.
  • Net Take-Home Calculation: The system takes the final calculated Gross salary and subtracts all the Employee liabilities: Employee PF, Employee ESI, Professional Tax, Labour Welfare Fund (LWF), and any advanced Bonus deductions.
  • Finish: It compiles all 65+ generated values (Basic, Allowances, Net Take Home, Taxes, etc.) into a refcursor table row, and hands the complete structure back to the requesting system.
