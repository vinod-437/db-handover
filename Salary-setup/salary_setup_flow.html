Explanation of the Tables & Key Fields Involved

  • empsalaryregister (The Primary Salary Table): Stores current active salary records. Key Operations: isactive = 0 (deactivates old), isactive = 1 (activates new). Key fields tracked include monthlyofferedpackage, breakdown values like basic and hra, and net outputs like ctc and salaryinhand.
  • openappointments (The Core Employee Record): Manages the employee's onboarding tracking and major compliance flags. Key fields updated include appointment_status_id, pfopted, esiopted, and setting the final offered_salary.
  • tbl_tpemp_leavetemplates (Leave Management): Maps the assigned leave policies, primarily updating clgranted, mlgranted, and totalleaves.
  • trn_candidate_otherduction (Pending External Deductions): Holds pending pre-calculated deductions (like meal vouchers). When the structure is saved, these are permanently linked by stamping them with the new salaryid.
  • users & PersonalInformation: During the final save stage, if this is a candidate conversion process, records here are spun up using the newly generated emp_code along with base demographic data.
